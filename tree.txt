Project Directory Structure: The Narrative Forge

This document outlines the standard directory structure for the project, aligning with the planned Epics. It separates source code by language, and isolates documentation, database assets, and runtime data.

/narrative-forge
├── **data/**
│   └── **archives/** # Storage for compressed chapter snapshots (Epic 6)
├── **docs/**
│   ├── database_architecture.md  # Documentation for the SQL schema (Epic 1)
│   └── project_directory_structure.md # This file
│   └── REVISED_PLAN_V2.md        # The main development roadmap
├── **sql/**
│   └── schema_v1.sql             # Complete database creation script (Epic 1)
├── **src/**
│   ├── **cpp/** # All C/C++ source code for performance modules
│   │   ├── **causality/** # C++ Chronology Checker (Epic 4)
│   │   │   └── causality_processor.cpp
│   │   ├── **text_indexer/** # C/C++ Inverted Index and Search (Epic 5)
│   │   │   └── indexer.c
│   │   └── **versioning/** # C SHA-256 and Snapshot Utility (Epic 6)
│   │       └── snapshot_engine.c
│   └── **python/** # All core Python/PyQt6 application logic
│       ├── **ui/** # PyQt6 GUI components and custom widgets
│       │   ├── chapter_editor.py
│       │   ├── lore_editor.py
│       │   └── tag_widget.py     # Reusable Tagging UI component (Epic 3)
│       ├── db_connector.py       # SQLite connection handler (Story 2.1)
│       ├── main_window.py        # Main PyQt6 application window (Story 2.2)
│       ├── outline_manager.py    # QTreeWidget logic (Story 2.3)
│       ├── causality_wrapper.py  # Python ctypes wrapper for C++ causality module (Epic 4)
│       └── __init__.py           # Makes 'python' a package
├── **tests/** # Unit and integration tests
└── README.md
